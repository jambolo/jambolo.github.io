Outputs to the debugger only in a DEBUG build.

When compiling a debug version, the trace function outputs to the debugger. The parameters and format string are the same as printf().

When compiling a release version, the trace call is optimized out. It is a clever hack. For example, the statement

trace( "%d\n", x );

is changed by the preprocessor to

0 && __BiteMe__( "%d\n", x );

The compiler optimizes the statement to nothing. The parameters are not evaluated because of C's early out functionality.

formatprintf-style format string
...parameters
Note
The size of the output is limited to 1024 bytes at a time.
If you are using MFC, you should use the MFC TRACE() functions, instead of trace().
